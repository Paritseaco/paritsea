# Paritsea — Language & Terminology Guide

This document records all language decisions for the Paritsea project. Reference this before writing any copy, translating content, or building new pages. Do not repeat decisions already made here.

---

## English Style

- **UK English** throughout — not US English.
  - Correct: *legitimacy, organised, recognised, behaviour, analyse, licence (n.) / license (v.)*
  - Not: *legitimacy, organized, recognized, behavior, analyze, license (n.)*
- Write in an **editorial, constitutional register** — precise, calm, unhurried.
- Avoid startup language: *leverage, unlock, empower, game-changer, seamless, scalable*.
- Avoid coaching language: *journey, growth, transform yourself, level up*.
- Avoid marketing superlatives: *best-in-class, world-class, cutting-edge*.

---

## Brand Archetype

**The Architect + The Witness**

- The Architect: builds structural systems with layers, hierarchy, and logic.
- The Witness: observes without rushing to judge; names tension before it becomes normalised.
- The brand does **not** sell, coach, or certify. It observes, defines, and makes available.

## Thai Language Principles

### Core Rule
**Do not translate directly from English.** Write Thai natively, keeping only the core message. Thai has different rhetorical structure — a sentence that works in English often reads as stiff or bureaucratic when translated word-for-word.

### Thai Sentence Structure
- Thai climactic structure: key concept lands **at the end**, not the front.
  - ✗ "ความสอดคล้องเชิงโครงสร้าง คือเงื่อนไขของความชอบธรรม" (front-loaded)
  - ✓ "ก่อนจะกระทำได้อย่างชอบธรรม โครงสร้างต้องสอดคล้องก่อน" (climactic)
- Use short, direct clauses. Thai readers parse S-V-O faster than long compound sentences.
- Avoid stacking modifiers before nouns: "กรอบแนวทางอ้างอิงเชิงสาธารณะแห่งความชอบธรรม" = unreadable.
- Avoid overusing คือ — each paragraph should have at most one.

### Thai Typography
- `font-family: Sarabun` (weight 300 for body/subheadings, 400 for display titles)
- `line-height: 1.85` (body), `1.4` (headings), `1.25` (hero/display)
- `letter-spacing: 0` always — Thai does not use letter-spacing
- `word-break: keep-all` — prevents break mid-syllable

### Concept Vocabulary

| English concept | Thai term | Notes |
|-----------------|-----------|-------|
| structural coherence | ความสอดคล้องเชิงโครงสร้าง | Keep — accurate, acceptable in intellectual Thai |
| structural tension | แรงตึงเชิงโครงสร้าง | Acceptable; can also use ความไม่ลงรอยในโครงสร้าง for clarity |
| legitimacy | ความชอบธรรม | Standard Thai, keep |
| normalised practice | บรรทัดฐาน | "norm/standard practice" — keep |
| accountability | ความรับผิด | More precise than ความรับผิดชอบ in structural context |
| applied judgment | การวินิจฉัยเชิงปฏิบัติ | Not การตัดสินใจ (too casual) |
| structural condition | สภาวะที่โครงสร้างกำหนด | Not "สภาวะเชิงโครงสร้าง" (passive, less clear) |
| constitutional (adjective) | **avoid** | Replace with แก่น, ฐาน, or รากฐาน depending on context |
| doctrine (general adjective) | แก่น- | e.g. "doctrinal authority" = อำนาจจากแก่น |
| reference framework | กรอบอ้างอิง | Drop "แนวทาง" prefix — กรอบอ้างอิง is sufficient |

### Words to Avoid in Thai

| Avoid | Reason | Use instead |
|-------|--------|-------------|
| รัฐธรรมนูญ | Sounds like national constitution/legal statute | แก่น, ฐาน |
| หลักคำสอน | Religious/dogmatic connotation | แก่น |
| กฎ | "Rule" — sounds restrictive | หลักการ, เงื่อนไข |
| บังคับใช้ | "Enforce" — sounds authoritarian when overused | นำไปใช้, ปฏิบัติตาม |
| ฟรี | "Free" — trivialises the framework | เปิดให้เข้าถึง, สาธารณะ |
| ใช้เข้าใจง่าย | Too casual/marketing | ใช้ได้จริง, นำไปประยุกต์ได้ |

## Hero Copy (Homepage)

The headline creates tension the reader immediately recognises — not a philosophical statement.
Both versions are written natively (Thai is NOT a translation of the English).

### English
```
The system works.
Something is still wrong.
```
Sub: *Paritsea is a public reference framework for naming structural tension before it becomes normalised practice.*

### Thai (native — not translated)
```
ระบบไม่ได้พัง
แต่บางอย่างผิดอยู่
```
Sub: *Paritsea ช่วยตั้งชื่อสิ่งที่ผิดในโครงสร้าง — ก่อนมันจะกลายเป็นบรรทัดฐาน*

### Why this works
- Two-line paradox: system functions ≠ system is legitimate
- Reader feels it before they understand it
- Mirrors the closing quote — gives the whole page coherence
- Avoids abstract philosophical framing ("structural coherence is a prerequisite...")

## Visual Language (for SVG / illustrations)

- Monospace numbers for layer labels: `01`, `02`, `03`, `04`, `05`
- Thin lines (`stroke-width: 0.75–1.75`) — never thick
- Accent colour for wave/sea elements: `#c4a882` (sand)
- Primary elements in `currentColor` — adapts to light/dark
- Wave as a threshold between theory (Standards) and practice (The Method)
- Anchor dot at key intersections — structural clarity point
- No decorative gradients, no glow effects, no drop shadows
