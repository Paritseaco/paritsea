# Paritsea Language Hierarchy System

## Overview

Paritsea is not a content platform.  
It is a cognitive language system that structures how thought is formed, reframed, and deepened over time.

Language is not UI decoration.  
Language is architecture of cognition.

This system defines a 3-layer vocabulary model that governs how meaning is introduced, transformed, and internalized.

---

# 1. Language Layers

## 1.1 Entry Vocabulary (Surface Layer)

### Purpose
Enable immediate comprehension without cognitive resistance.

### Function
- Onboarding understanding
- Low-friction reading
- Emotional and intuitive accessibility

### Vocabulary
- Explore Journal
- Reflection
- Thinking Model

### Rules
- Must be instantly understandable
- No abstract or academic terminology
- If interpretation is required → NOT allowed here

### Constraint
> If a user must “analyze the word” before continuing, it does not belong in this layer.

---

## 1.2 Gateway Vocabulary (Transition Layer)

### Purpose
Shift the user's frame of thinking without explicit instruction.

### Function
- Cognitive reframing
- Perspective disruption
- Subtle model transition

### Vocabulary
- Mindset
- Approach
- Perspective Shift

### Rules
- Must feel familiar but subtly transform meaning
- Should not require explanation to be effective
- Acts as a bridge between surface and core cognition

### Constraint
> User should feel understanding, but later realize meaning has deepened.

---

## 1.3 Foundational Ontology (Core Layer)

### Purpose
Define the structural reality of how thinking works inside Paritsea.

### Function
- System architecture of thought
- Deep conceptual modeling
- Framework-level reasoning

### Vocabulary
- Cognitive Pattern
- Recursive Thought Structure
- Perception Architecture

### Rules
- Not required for immediate understanding
- Must be consistent and precise across system
- Used in deep content, system logic, and framework explanation

### Constraint
> Core vocabulary is not for onboarding. It is for reconstruction of cognition.

---

# 2. Semantic Progression Rule

Language in Paritsea is progressive.

Users should naturally move:

Entry → Gateway → Core

This is not optional. It is structural.

### Progression Principle
- Entry introduces perception
- Gateway reshapes interpretation
- Core redefines structure of thought

---

# 3. Cognitive Load Rule

- Entry: minimize cognitive load
- Gateway: controlled ambiguity
- Core: high precision, low accessibility

---

# 4. Forbidden Misuse Rules

## Do NOT:
- Use Core vocabulary in UI onboarding
- Expose ontology terms without context
- Skip Gateway layer in user-facing flow

---

# 5. Design Principle

Paritsea does not explain thinking.

It structures thinking through language exposure over time.
